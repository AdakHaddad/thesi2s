\section{Ringkasan Sistem}

Bab ini menjelaskan implementasi peripheral \textit{I2S transmitter} berbasis arsitektur \textit{Direct Digital Synthesis} (DDS) yang dirancang sebagai custom IP AXI4-Lite. Fokus rancangan adalah mengubah sampel PCM stereo yang ditulis oleh MicroBlaze V melalui register menjadi sinyal serial \path{i2s_mclk}, \path{i2s_bclk}, \path{i2s_ws}, dan \path{i2s_data} dengan fleksibilitas frekuensi sampling yang dapat diprogram secara arbitrer. Implementasi utama merujuk pada desain \path{latesti2s} (\path{i2s_dds_1_0}).

\section{Spesifikasi IP I2S}

\subsection{Fitur utama}
\begin{itemize}
    \item Antarmuka kontrol \textbf{AXI4-Lite 32 bit} untuk akses dari MicroBlaze V.
    \item Pembangkitan detak audio murni digital (\textbf{DDS/NCO}), bersifat \textbf{vendor-agnostic}.
    \item Mendukung \textbf{frekuensi sampling arbitrer} yang dapat diatur melalui register dalam satuan Hz.
    \item Format \textbf{Philips I2S} standar (stereo, MSB-first, one-bit delay).
    \item Lebar sampel dinamis (1--32 bit) dan mekanisme \textit{shadow registers} untuk integritas data.
    \item Mekanisme \textit{handshake} melalui register \textbf{STATUS} untuk sinkronisasi software-hardware.
\end{itemize}

\section{Implementasi RTL IP I2S DDS}

Implementasi RTL menggunakan akumulator fase yang berjalan pada domain 100 MHz.

\subsection{Pembangkitan Clock Berbasis Modulus NCO}

Desain ini menggunakan akumulator modulus terhadap $100.000.000$. Setiap kali akumulator meluap, register detak berpindah logika (\textit{toggle}).

\begin{lstlisting}[language=Verilog, caption={Logika mesin DDS pada \texttt{i2s\_dds\_slave\_lite\_v1\_0\_S00\_AXI.v}.}]
wire [31:0] bclk_toggle_rate_hz = {sample_rate_sync, 7'b0}; // Fs x 128
wire bclk_phase_wrap = (bclk_phase_sum >= AXI_CLK_HZ);

always @(posedge S_AXI_ACLK) begin
    if (bclk_phase_wrap) begin
        bclk_phase_acc_q <= bclk_phase_sum - AXI_CLK_HZ;
        bclk_q <= !bclk_q;
    end else begin
        bclk_phase_acc_q <= bclk_phase_sum;
    end
end
\end{lstlisting}

\subsection{Serializer dan Sinkronisasi Sampel}

Logika serialisasi menggunakan fungsi \texttt{i2s\_next\_serial\_bit} untuk menangani urutan bit MSB-first. Pengambilan sampel dari register AXI dilakukan ke dalam \textit{shadow registers} tepat pada awal frame stereo (\texttt{bit\_count == 0}).

\begin{lstlisting}[language=Verilog, caption={Latching sampel ke shadow registers pada tepi naik BCLK.}]
if (bclk_q == 1'b0) begin
    bclk_q <= 1'b1;
    if (bit_count_q == 6'd0) begin
        sample_left_q          <= slv_reg0;
        sample_right_q         <= slv_reg1;
        current_latch_status_q <= !current_latch_status_q;
    end
end
\end{lstlisting}

Perpindahan logika \texttt{current\_latch\_status\_q} diekspor ke register \texttt{STATUS}, memungkinkan perangkat lunak untuk mengetahui kapan hardware telah mengonsumsi sampel sebelumnya.

\section{Implementasi Software Vitis}

Perangkat lunak pada Vitis (\path{latesti2s/ready_to_run/sw/helloworld.c}) dirancang untuk menjadi sumber data audio yang fleksibel dan interaktif. Berbeda dengan implementasi sederhana, versi final ini mengadopsi kontrol \textit{closed-loop} terhadap status perangkat keras dan antarmuka pengguna berbasis UART.

\subsection{Definisi Alamat dan Masker Bit}

Aplikasi mendefinisikan konstanta alamat register yang sesuai dengan peta register IP I2S DDS.

\begin{lstlisting}[language=C, caption={Konfigurasi register dan masker bit pada software.}]
#define DATA_LEFT   (I2S_BASE + 0x00U)
#define DATA_RIGHT  (I2S_BASE + 0x04U)
#define CONTROL     (I2S_BASE + 0x08U)
#define STATUS_REG  (I2S_BASE + 0x0CU)

#define CTRL_ENABLE      (1U << 0)
#define CTRL_MUTE        (1U << 1)
#define CTRL_WIDTH(w)    (((uint32_t)((w) >= 32U ? 0U : ((w) & 0x1FU))) << 2U)
#define CTRL_RATE(hz)    (((uint32_t)(hz) & 0x000FFFFFU) << 7U)
\end{lstlisting}

\subsection{Logika Validasi dan \textit{Clamping} Parameter}

Untuk menjamin stabilitas sistem, perangkat lunak melakukan validasi terhadap setiap perubahan parameter. Fungsi \texttt{clamp\_sample\_rate} memastikan frekuensi sampling tidak melebihi batas aman Nyquist (dikap pada 195.312 Hz untuk sistem clock 100 MHz), sementara \texttt{clamp\_width} memastikan lebar bit berada pada rentang 1--32 bit.

\subsection{Antarmuka Pengguna Interaktif (UART)}

Perangkat lunak mengimplementasikan penanganan \textit{asynchronous input} melalui terminal serial. Pengguna dapat mengubah parameter secara langsung dengan menekan tombol pada keyboard.

\begin{lstlisting}[language=C, caption={Penanganan input UART untuk kontrol interaktif.}]
void handle_key(uint8_t key) {
    switch (tolower(key)) {
        case 'e': set_enable(!g_enable); break;
        case 'm': set_mute(!g_mute);     break;
        case 'w': set_width(g_width == 16 ? 24 : g_width == 24 ? 32 : 16); break;
        case 'f': set_sample_rate(prompt_value("New Fs (Hz): ")); break;
        case 't': set_tone_freq(prompt_value("New Tone (Hz): "));  break;
    }
}
\end{lstlisting}

\subsection{Pembangkitan Gelombang DDS dan Sinkronisasi Frame}

Inti dari proses pengiriman audio terletak pada loop utama yang menunggu perubahan bit status dari hardware. Ketika hardware melakukan \textit{latch} pada sampel sebelumnya, software segera menghitung dan menulis sampel berikutnya.

\begin{lstlisting}[language=C, caption={Loop utama pengiriman sampel dengan sinkronisasi status hardware.}]
while (1) {
    // 1. Polling bit status untuk sinkronisasi frame
    while (frame_status_bit() == g_last_frame_bit) {
        // Cek input UART tanpa mengganggu audio
        if (!XUartLite_IsReceiveEmpty(STDIN_BASEADDRESS))
            handle_key(XUartLite_RecvByte(STDIN_BASEADDRESS));
    }
    g_last_frame_bit = frame_status_bit();

    // 2. Hitung sampel sinus/square menggunakan DDS software
    stream_software_dds_sample();
}
\end{lstlisting}

Teknik DDS software menggunakan akumulator fase 32-bit (\path{g_phase}) yang bertambah sebesar \path{step} setiap sampel. Nilai \path{step} dihitung berdasarkan rasio frekuensi nada terhadap frekuensi sampling target. Hal ini memungkinkan perubahan nada yang sangat halus dan akurat tanpa memerlukan tabel gelombang yang sangat besar.

\subsection{Timing Philips I2S}

Sesuai standar Philips, transisi \texttt{WS} terjadi satu periode \texttt{BCLK} sebelum MSB data. Implementasi RTL menjamin relasi ini dengan memperbarui \texttt{WS} dan \texttt{DATA} pada sisi turun (\textit{falling edge}) \texttt{BCLK}.

\section{Ringkasan Bab}

Implementasi final menggunakan arsitektur DDS murni digital yang menawarkan presisi frekuensi tinggi dan kemudahan integrasi. Mekanisme \textit{shadow register} dan \textit{status polling} menjamin integritas transmisi audio stereo pada berbagai \textit{sampling rate} yang dapat diprogram.
