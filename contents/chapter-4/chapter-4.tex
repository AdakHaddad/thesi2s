\section{Implementasi RTL (Verilog)}

\subsection{Struktur Modul}

Implementasi RTL terdiri dari modul-modul berikut:

\begin{itemize}
    \item \texttt{i2s\_top.v}: Modul tingkat atas yang menghubungkan antarmuka
          AXI4-Lite, logika register, FIFO TX, serializer I2S, pembangkit clock,
          dan logika IRQ.
    \item \texttt{i2s\_axi\_slave.v}: Slave AXI4-Lite; menangani \textit{handshake}
          AWVALID/AWREADY, WVALID/WREADY, BVALID/BREADY, ARVALID/ARREADY,
          RVALID/RREADY, dan mendekode alamat ke register internal.
    \item \texttt{i2s\_fifo.v}: Ring buffer 256 entri $\times$ 48-bit; antarmuka
          \textit{push}/\textit{pop} satu siklus.
    \item \texttt{i2s\_serializer.v}: Mesin \textit{state} pembangkit BCLK/WS/DATA
          berdasarkan $audio\_clk$.
    \item \texttt{i2s\_clkdiv.v}: Pembagi clock untuk BCLK (mode 96k: ÷4;
          mode 48k: ÷8) dengan \textit{duty cycle} 50\%.
\end{itemize}

\subsection{Parameter AXI4-Lite}

Pada modul \texttt{i2s\_top.v}, parameter AXI4-Lite disesuaikan dari
\textit{template} Vivado sebagai berikut:

\begin{itemize}
    \item \texttt{C\_S00\_AXI\_ADDR\_WIDTH = 8}: Ruang alamat 256 byte (cukup
          untuk 9 register $\times$ 4 byte).
    \item \texttt{OPT\_MEM\_ADDR\_BITS = 3}: Mendukung dekode 16 register
          (\textit{word index} 4-bit, offset 2 bit).
    \item \texttt{C\_S00\_AXI\_DATA\_WIDTH = 32}: Data 32-bit.
\end{itemize}

\subsection{Implementasi Pembagi Clock}

Pembagi clock diimplementasikan dengan pencacah mod-$D/2$ yang men-\textit{toggle}
sinyal BCLK setiap $D/2$ siklus $audio\_clk$:

\begin{verbatim}
// i2s_clkdiv.v (ringkasan)
always @(posedge audio_clk or posedge rst) begin
    if (rst) begin
        cnt    <= 0;
        bclk_r <= 0;
    end else if (cnt == half_div - 1) begin
        cnt    <= 0;
        bclk_r <= ~bclk_r;  // toggle: duty 50%
    end else begin
        cnt <= cnt + 1;
    end
end
// half_div = 2 (mode 96k, total div 4)
// half_div = 4 (mode 48k, total div 8)
\end{verbatim}

\subsection{Implementasi FIFO TX}

\begin{verbatim}
// i2s_fifo.v (ringkasan)
reg [47:0] mem [0:255];
reg [7:0]  wr_ptr, rd_ptr;

assign full  = (wr_ptr + 1) == rd_ptr;
assign empty = (wr_ptr == rd_ptr);
assign level = wr_ptr - rd_ptr;   // modulo 256

always @(posedge clk) begin
    if (push && !full)
        mem[wr_ptr] <= {left_in, right_in};
    if (push && !full) wr_ptr <= wr_ptr + 1;
end

always @(posedge clk) begin
    if (pop && !empty) begin
        {left_out, right_out} <= mem[rd_ptr];
        rd_ptr <= rd_ptr + 1;
    end
end
\end{verbatim}

\section{Pengujian Fungsional (Simulasi RTL)}

\subsection{Testbench}

\textit{Testbench} \texttt{tb\_i2s\_top.v} mensimulasikan:
\begin{enumerate}
    \item Transaksi AXI4-Lite untuk menulis register CONTROL, WATERMARK,
          TX\_LEFT, dan TX\_RIGHT.
    \item Pengisian FIFO hingga 10 \textit{frame} dengan pola data
          \textit{ramp} (0x000001, 0x000002, \ldots).
    \item Aktivasi ENABLE dan pemantauan sinyal BCLK, WS, DATA selama
          beberapa \textit{frame}.
    \item Pemantauan IRQ saat level FIFO mencapai \textit{watermark}.
\end{enumerate}

\subsection{Hasil Verifikasi Simulasi}

Dari simulasi, hal-hal berikut diverifikasi:
\begin{itemize}
    \item WS \textit{toggle} setiap 32~BCLK; tidak ada \textit{glitch}.
    \item DATA MSB-first, dimulai satu BCLK setelah \textit{edge} WS
          (1-bit \textit{delay} format Philips I2S terpenuhi).
    \item FIFO \textit{level} berkurang 1 setiap \textit{frame}; IRQ muncul
          saat level = WATERMARK.
    \item Kondisi \textit{underrun}: ketika FIFO kosong, DATA = 0 dan
          bit UNDERRUN pada STATUS diset.
\end{itemize}

Pengujian audio memverifikasi:
\begin{itemize}
    \item \textit{Tone} 1~kHz terdengar jelas tanpa \textit{clicking} atau
          \textit{popping} yang disebabkan \textit{underrun}.
    \item Mekanisme ISR FIFO \textit{refill} beroperasi tanpa \textit{underrun}
          saat CPU menjalankan logika pembangkitan \textit{sine wave}.
    \item Pergantian mode $F_s$ (48k $\to$ 96k dan sebaliknya) tidak menyebabkan
          \textit{glitch} audio saat urutan reset yang benar diikuti.
\end{itemize}
