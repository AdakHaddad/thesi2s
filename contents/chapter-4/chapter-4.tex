\chapter{Implementasi dan Pengujian}

\section{Implementasi RTL (Verilog)}

Peripheral I\textsuperscript{2}S TX diimplementasikan dalam bahasa Verilog dengan struktur modul sebagai berikut:

\begin{itemize}
    \item \textbf{\texttt{i2s\_tx\_top.v}}: Modul \textit{top-level} yang menggabungkan AXI slave, FIFO TX, serializer I\textsuperscript{2}S, dan logika IRQ.
    \item \textbf{\texttt{i2s\_axi\_slave.v}}: Modul AXI4-Lite slave yang menangani transaksi baca-tulis register sesuai peta register pada Tabel~\ref{tab:register_map}.
    \item \textbf{\texttt{i2s\_fifo\_tx.v}}: Modul FIFO TX sinkron berbasis BRAM (256 frame stereo, 48-bit lebar data).
    \item \textbf{\texttt{i2s\_serializer.v}}: Modul serializer I\textsuperscript{2}S yang membangkitkan BCLK, WS, dan DATA sesuai format Philips I\textsuperscript{2}S 32-bit slot.
    \item \textbf{\texttt{i2s\_irq\_ctrl.v}}: Modul logika IRQ untuk watermark dan underrun.
\end{itemize}

\noindent\textbf{Catatan implementasi kunci}:
\begin{itemize}
    \item \texttt{C\_S\_AXI\_ADDR\_WIDTH} disetel ke 8-bit (mendukung 64 alamat byte / 16 register 32-bit).
    \item BCLK dibangkitkan dari \texttt{audio\_clk} menggunakan counter pembagi; pembagi dipilih berdasarkan \texttt{FS\_MODE}.
    \item Satu bit delay pada DATA diimplementasikan dengan menggeser (shift) output serializer satu BCLK.
    \item Padding 7-bit nol setelah bit data ke-24 ditangani oleh logika \textit{bit\_counter} di serializer.
    \item Sinyal \texttt{i2s\_mclk} tersedia sebagai output (dihubungkan ke \texttt{audio\_clk}) meskipun tidak selalu diperlukan oleh PCM5102A.
\end{itemize}

\section{Pengujian Fungsional melalui Simulasi RTL}

Simulasi RTL dilakukan menggunakan Vivado Simulator dengan testbench yang mengeksitasi antarmuka AXI4-Lite untuk mensimulasikan penulisan data audio (pola \textit{ramp} atau gelombang sinus 1~kHz).

Prosedur pengujian simulasi:
\begin{enumerate}
    \item Inisialisasi: tulis CONTROL (ENABLE=0, FIFO\_RST=1), tunggu.
    \item Set WATERMARK = 64, IRQ\_EN = 0x01.
    \item Prefill: tulis 200 frame (TX\_LEFT + TX\_RIGHT) ke FIFO.
    \item Set CONTROL[ENABLE]=1.
    \item Amati sinyal: BCLK, WS, DATA, FIFO\_LEVEL, IRQ.
    \item Verifikasi: WS berganti setiap 32 BCLK, DATA sesuai MSB-first 24-bit, 1-bit delay.
\end{enumerate}

Aspek yang diverifikasi dari simulasi:
\begin{itemize}
    \item WS berganti tepat setiap 32 BCLK (konfirmasi frame 64 BCLK).
    \item DATA dimulai satu siklus BCLK setelah transisi WS (1-bit delay).
    \item Bit ke-1 hingga ke-24 sesuai dengan nilai data 24-bit yang ditulis.
    \item Bit ke-25 hingga ke-32 adalah nol (padding).
    \item IRQ aktif saat FIFO\_LEVEL $\le$ WATERMARK.
    \item Setelah FIFO kosong: DATA = 0x000000 (underrun handling).
\end{itemize}

Pengujian audio mencakup:
\begin{enumerate}
    \item Playback gelombang sinus 1~kHz: verifikasi frekuensi output sesuai pada oscilloscope.
    \item Pengujian mute: set \texttt{CONTROL[MUTE]=1}, output harus nol/senyap.
    \item Pengujian mode Fs: ganti antara Mode~0 ($\approx$48~kHz) dan Mode~1 ($\approx$96~kHz) menggunakan prosedur aman (pause $\to$ reset FIFO $\to$ ubah mode $\to$ prefill $\to$ play).
    \item Pengujian underrun: hentikan pengisian FIFO, amati gangguan (\textit{glitch}) pada output audio dan flag UNDERRUN pada ILA.
    \item Pengujian watermark IRQ: amati respon ISR saat IRQ aktif dan keberhasilan pengisian ulang FIFO.
\end{enumerate}

\subsection{Overhead CPU: Polling vs.\ Interrupt}

Mekanisme \textit{interrupt watermark} secara signifikan mengurangi beban CPU dibandingkan \textit{polling} terus-menerus. Dengan watermark = 64 frame pada Mode~1 ($F_s \approx 96$~kHz), ISR terpicu kira-kira setiap:

\[
t_{ISR} = \frac{(256 - 64) \text{ frame}}{95{,}97 \times 10^3 \text{ frame/s}} \approx 2{,}0\,\text{ms}
\]

Ini berarti prosesor bebas selama $\sim$2~ms antara setiap ISR, cukup untuk menjalankan tugas lain tanpa risiko underrun (selama ISR mengisi ulang dengan cepat).
