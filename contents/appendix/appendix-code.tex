\subsection{Struktur modul utama}

\begin{enumerate}
    \item \textbf{AXI4-Lite register file}: menerima transaksi tulis/baca dari MicroBlaze V dan menyimpan empat register, yaitu \texttt{DATA\_LEFT}, \texttt{DATA\_RIGHT}, \texttt{CONTROL}, dan \texttt{STATUS}.
    \item \textbf{DDS phase accumulator}: membangkitkan tepi \texttt{BCLK} dan \texttt{MCLK} dari clock sistem 100 MHz berdasarkan nilai \texttt{SAMPLE\_RATE\_HZ}.
    \item \textbf{Sample latch}: mengambil sampel kiri-kanan dari register AXI ke shadow register pada awal frame stereo (\texttt{bit\_count == 0}).
    \item \textbf{STATUS toggle}: bit 0 pada \texttt{STATUS} berubah setiap latch sehingga firmware dapat melakukan \textit{polling}.
    \item \textbf{Serializer}: mengubah sampel paralel menjadi aliran bit Philips I2S dengan satu bit delay, MSB-first, dan zero padding.
    \item \textbf{Output register}: menggerakkan \texttt{i2s\_mclk}, \texttt{i2s\_bclk}, \texttt{i2s\_ws}, dan \texttt{i2s\_data}.
\end{enumerate}

\section{Kode Aplikasi MicroBlaze}

Berkas aplikasi berada pada:

\begin{lstlisting}[caption={Lokasi kode aplikasi Vitis.}]
ready_to_run/sw/helloworld.c
\end{lstlisting}

Aplikasi ini menjalankan pengujian bare-metal terhadap IP I2S. Firmware
membangkitkan gelombang uji (sinus/kotak) di sisi software, memprogram
\texttt{SAMPLE\_RATE\_HZ} dan \texttt{SAMPLE\_WIDTH} melalui register
\texttt{CONTROL}, lalu menulis sampel ke register data dengan sinkronisasi
berbasis \textit{polling} pada \texttt{STATUS[0]}.

\section{Skrip Rekonstruksi Proyek Vivado}

Proyek Vivado dapat direkonstruksi menggunakan skrip Tcl yang tersedia pada
repositori proyek:

\begin{lstlisting}[caption={Lokasi skrip rekonstruksi proyek Vivado.}]
ready_to_run/fsi2snfilter.tcl
ready_to_run/rebuild.tcl
\end{lstlisting}

Skrip ini membangun ulang block design \texttt{bdesign\_clock\_core} dengan IP
\path{i2s_0}, MicroBlaze V, AXI SmartConnect, UARTLite, dan peripheral
pendukung. Hubungan antar komponen dijaga agar proyek dapat dibuat ulang tanpa
bergantung pada langkah manual di antarmuka Vivado.

Bab III menjelaskan metode perancangan dan verifikasi. Bab IV menjelaskan
arsitektur dan potongan kode inti dari RTL dan firmware. Bab V menunjukkan
hasil pengukuran sinyal serta validasi audio pada perangkat keras. Dengan
pembagian ini, kode sumber tidak hanya disimpan sebagai lampiran, tetapi juga
dikaitkan langsung dengan desain, metode pengujian, dan hasil akhir peripheral
I2S.
