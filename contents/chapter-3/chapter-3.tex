\section{Metode yang Digunakan}

\subsection{Metode perancangan hardware}

Perancangan perangkat keras dilakukan dengan tahapan berikut:
\begin{enumerate}
    \item Menentukan spesifikasi I2S TX meliputi format data~\citep{philips_i2s_1986}, lebar sampel, mode frekuensi sampling, dan pemetaan register.
    \item Mendesain arsitektur modul yang mencakup AXI4-Lite slave~\citep{arm_amba_axi_2010}, register data dan kontrol, pembagi clock, serta serializer I2S.
    \item Melakukan optimasi desain menggunakan teknik DDS untuk mendukung frekuensi sampling arbitrer dan membuat IP bersifat \textit{vendor-agnostic}.
    \item Mengimplementasikan RTL dalam Verilog~\citep{palnitkar_verilog_2003} dengan pemisahan blok sekuensial dan kombinasional secara jelas.
    \item Mengintegrasikan peripheral ke dalam Vivado IP Integrator~\citep{xilinx_vivado_design_suite_2021} bersama MicroBlaze V, BRAM, clocking, dan peripheral pendukung.
\end{enumerate}

\noindent Alur tersebut sejalan dengan pendekatan pengembangan sistem FPGA embedded
yang menempatkan definisi spesifikasi, implementasi RTL, integrasi IP, dan
validasi perangkat lunak sebagai tahapan yang saling terkait, bukan pekerjaan
yang berdiri sendiri~\citep{lafosse_embedded_2018}.

\subsection{Metode verifikasi dan simulasi}

Verifikasi dilakukan pada dua level:
\begin{enumerate}
    \item \textbf{Validasi hardware}, untuk memastikan desain tetap bekerja setelah \textit{synthesis} dan \textit{implementation} pada FPGA nyata.
    \item \textbf{Verifikasi fungsional RTL}, untuk mereproduksi perilaku sinyal I2S, urutan bit, dan logika \texttt{DATA}/\texttt{CONTROL} secara terkontrol melalui simulasi.
\end{enumerate}

Parameter yang diamati meliputi:
\begin{itemize}
    \item relasi timing \texttt{i2s\_bclk}, \texttt{i2s\_ws}, dan \texttt{i2s\_data};
    \item kontinuitas aliran sampel pada skenario uji;
    \item akurasi frekuensi sampling nominal rendah dan tinggi;
    \item keberhasilan keluaran audio pada DAC.
\end{itemize}

\noindent Suatu pengujian dinyatakan lulus apabila empat syarat utama terpenuhi, yaitu:
struktur frame terdiri dari 64 pulsa BCLK per sampel stereo, transisi \texttt{WS} terjadi satu
BCLK sebelum MSB kanal berikutnya, bit tunda setelah transisi \texttt{WS} bernilai nol, dan
DAC mampu menghasilkan audio yang konsisten dengan mode kanal serta frekuensi yang dipilih.
Dengan definisi ini, verifikasi tidak berhenti pada kecocokan waveform semata, tetapi juga
menilai apakah perilaku sistem sudah sesuai dengan tujuan operasional peripheral.

\noindent Pada alur kerja aktual penelitian ini, verifikasi perilaku dasar terlebih dahulu
dilakukan melalui \textit{bitstream bring-up}, simulasi \textit{testbench}, dan pengukuran pada perangkat
keras. Setelah hubungan timing utama telah teridentifikasi pada FPGA nyata, dibuat
\textit{testbench} Vivado/XSim pada proyek uji
untuk mereproduksi kondisi \texttt{enable}, \texttt{mute}, pemrograman \texttt{SAMPLE\_RATE\_HZ}, dan
serialisasi sampel kiri-kanan secara lebih formal. Dengan demikian, simulasi pada penelitian ini
berfungsi sebagai verifikasi pelengkap yang mendokumentasikan ulang perilaku yang telah diamati
pada perangkat keras, bukan sebagai satu-satunya sumber validasi.

\subsection{Metode integrasi SoC}

Integrasi peripheral ke dalam SoC dilakukan melalui alur berikut:
\begin{enumerate}
    \item \textbf{IP Packaging}: mengemas modul I2S sebagai custom IP menggunakan Vivado IP Integrator~\citep{xilinx_vivado_design_suite_2021}.
    \item \textbf{Block Design}: menghubungkan MicroBlaze V~\citep{xilinx_microblaze_ref_2021}, AXI interconnect~\citep{xilinx_axi_ref_2017}, BRAM, Clocking Wizard~\citep{xilinx_clocking_wizard_2021}, UART, dan IP I2S.
    \item \textbf{Address Allocation}: menetapkan base address peripheral agar dapat diakses dari Vitis~\citep{xilinx_vitis_embedded_2021}.
    \item \textbf{Constraint Management}: memetakan pin I2S ke konektor board sesuai file \texttt{.xdc}~\citep{xilinx_basys3_ref}.
\end{enumerate}

\subsection{Metode pengembangan software}

Perangkat lunak uji dikembangkan dalam bahasa C pada Vitis dengan fungsi utama:
\begin{itemize}
    \item menginisialisasi register peripheral I2S melalui antarmuka AXI4-Lite;
    \item menyinkronkan pengiriman sampel audio menggunakan mekanisme \textit{polling} pada register \texttt{STATUS} (\textit{hardware-software handshake});
    \item menghasilkan sinyal uji secara digital menggunakan teknik DDS pada sisi software untuk fleksibilitas bentuk gelombang;
    \item menyediakan antarmuka pengguna interaktif melalui terminal serial (UART) untuk mengubah parameter operasional secara waktu nyata (\textit{runtime});
    \item mengimplementasikan logika \textit{clamping} dan validasi untuk memastikan parameter \textit{sampling rate} dan lebar bit tetap berada dalam batas aman perangkat keras.
\end{itemize}

\section{Perancangan Block Design Sistem}

Perancangan \textit{Block Design} dilakukan untuk mengintegrasikan custom IP I2S
dengan sistem embedded berbasis MicroBlaze V. Tahapan ini diperlukan agar
peripheral yang dirancang pada level RTL dapat diakses melalui bus AXI4-Lite,
dikendalikan oleh perangkat lunak, serta dihubungkan ke pin keluaran FPGA.
Dokumentasi berkas sumber dan skrip rekonstruksi proyek disajikan pada
Lampiran, sedangkan bagian ini menjelaskan alur perancangannya secara runtut.

\subsection{Pembuatan Project Vivado}

Tahap awal dilakukan dengan membuat project baru pada Vivado sebagai RTL
Project. Board FPGA dipilih sesuai perangkat yang digunakan, yaitu Basys3
berbasis Artix-7. Setelah project dibuat, source Verilog untuk custom IP I2S
ditambahkan ke dalam lingkungan kerja Vivado agar dapat dikemas dan
diintegrasikan ke dalam sistem.

\subsection{Perancangan Block Design}

Block Design dibuat menggunakan Vivado IP Integrator. Pada tahap ini, komponen
sistem yang diperlukan dihubungkan dalam satu rancangan, meliputi prosesor
soft-core, memori lokal, interkoneksi AXI, modul reset, Clocking Wizard, UART,
serta custom IP I2S. Integrasi antarkomponen dilakukan dengan memperhatikan
kebutuhan antarmuka AXI4-Lite, domain clock, dan jalur reset agar seluruh IP
dapat bekerja dalam satu sistem yang konsisten.

\subsection{Integrasi MicroBlaze}

MicroBlaze V digunakan sebagai prosesor utama untuk mengendalikan peripheral I2S
melalui transaksi register. Konfigurasi sistem mencakup penambahan local memory
berbasis BRAM, koneksi bus instruksi dan data, konfigurasi Processor System
Reset, serta integrasi AXI SmartConnect sebagai jalur komunikasi antara prosesor
dan peripheral. Dengan konfigurasi ini, perangkat lunak yang berjalan pada
MicroBlaze dapat melakukan akses baca dan tulis ke alamat register IP I2S.

\subsection{Perancangan Clocking Audio Berbasis DDS}

Rancangan final pada penelitian ini menggunakan pendekatan \textit{Direct Digital Synthesis} (DDS) murni untuk membangkitkan seluruh detak audio. Seluruh logika berjalan pada domain detak sistem \texttt{S\_AXI\_ACLK} 100 MHz. Penggunaan DDS menghilangkan ketergantungan pada blok PLL/MMCM eksternal untuk frekuensi sampling audio, sehingga membuat IP bersifat \textit{vendor-agnostic} dan sangat fleksibel.

DDS bekerja dengan akumulator fase yang bertambah setiap siklus detak 100 MHz. Ketika akumulator meluap (\textit{wrap}), sebuah tepi detak (\textit{clock edge}) dibangkitkan. Hubungan antara frekuensi sampling $F_s$, detak sistem $f_{sys}$, dan nilai inkremen fase $\Delta \theta$ dinyatakan sebagai:

\begin{equation}
    \Delta \theta_{BCLK} = \frac{F_s \times 64 \times 2}{f_{sys}} \times \text{Modulus}
\end{equation}

Dalam implementasi Verilog pada \path{i2s.v}, digunakan akumulator modulus terhadap nilai $100.000.000$. Hal ini memungkinkan pembangkitan frekuensi yang sangat presisi karena pembagi disesuaikan langsung dengan jumlah siklus clock sistem per detik. Desain ini membangkitkan dua detak utama:
\begin{itemize}
    \item \textbf{BCLK}: Berjalan pada $64 \times F_s$, digunakan untuk serialisasi data.
    \item \textbf{MCLK}: Berjalan pada $256 \times F_s$, disediakan untuk modul DAC yang membutuhkan detak master eksternal.
\end{itemize}

\noindent Fitur unik pada rancangan ini adalah register \texttt{STATUS} yang memungkinkan perangkat lunak melakukan sinkronisasi dengan perangkat keras melalui mekanisme \textit{polling}. Setiap kali IP mengambil (\textit{latch}) sampel baru dari register data ke dalam \textit{shadow registers} internal di awal frame stereo, bit 0 pada register \texttt{STATUS} akan berubah logika (\textit{toggle}). Dengan memantau perubahan bit ini, perangkat lunak dapat menjamin bahwa data yang ditulis tidak terlewat atau tertumpuk.

\subsection{Implementasi Signal Generator}

Signal generator digital diimplementasikan pada sisi perangkat lunak sebagai
sumber sampel uji. Program menghasilkan sampel audio periodik, kemudian menulis
nilai tersebut ke register kanal kiri dan kanal kanan melalui AXI4-Lite.
Pengaturan parameter seperti frekuensi nada, kanal aktif, mode mute, dan mode
frekuensi sampling dilakukan melalui register kontrol sehingga respons IP dapat
diamati pada sinyal keluaran maupun keluaran audio DAC.

\subsection{Konfigurasi Constraint FPGA}

Constraint FPGA ditulis dalam file \texttt{.xdc} untuk memetakan sinyal I2S ke
pin FPGA yang terhubung dengan modul DAC. Selain penentuan pin, konfigurasi
\textit{I/O standard} juga ditetapkan agar sesuai dengan level tegangan pada
board dan modul eksternal. Tahap ini memastikan keluaran \texttt{MCLK},
\texttt{BCLK}, \texttt{LRCLK}, dan \texttt{SDATA} dapat diamati dan digunakan
oleh DAC.

\subsection{Validasi dan Generate Bitstream}

Setelah Block Design selesai, dilakukan validasi rancangan untuk memastikan
tidak ada koneksi atau konfigurasi IP yang bermasalah. Tahapan selanjutnya
adalah synthesis, implementation, dan generate bitstream. Bitstream yang
dihasilkan digunakan untuk memprogram FPGA sehingga rancangan dapat diuji pada
perangkat keras.

\subsection{Export Hardware ke Vitis}

Setelah bitstream berhasil dibuat, hardware platform diekspor ke Vitis dalam
bentuk file \texttt{.xsa}. File ini memuat informasi perangkat keras, address
map, konfigurasi prosesor, serta peripheral yang tersedia. Platform tersebut
kemudian digunakan sebagai dasar pembuatan aplikasi embedded untuk mengakses
register AXI, mengatur mode operasi IP I2S, dan membangkitkan sampel audio uji.
