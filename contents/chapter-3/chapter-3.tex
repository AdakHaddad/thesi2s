\section{Spesifikasi dan arsitektur IP I2S TX}

Bagian ini merangkum \textbf{perancangan fungsional} IP yang menjadi inti skripsi. Implementasi difokuskan pada peripheral \textbf{I2S transmitter} dengan antarmuka \textbf{AXI4-Lite} agar dapat dikontrol oleh program C pada MicroBlaze V.

\subsection{Daftar fitur IP}
\begin{itemize}
    \item Antarmuka \textbf{AXI4-Lite} 32 bit untuk konfigurasi dan pengisian data audio.
    \item Mode \textbf{TX-only}, stereo, format \textbf{Philips I2S}, \textbf{24 bit} per saluran dalam \textbf{slot 32 bit}.
    \item Dukungan dua keluarga frekuensi sampling nominal (\textbf{48/96 kHz} dan \textbf{44.1/88.2 kHz}), melalui pembagi clock internal.
    \item \textbf{Jalur data} lewat register \texttt{DATA\_LEFT}/\texttt{DATA\_RIGHT} dan \texttt{CONTROL}, \textbf{tanpa FIFO} internal.
    \item Keluaran menuju modul \textbf{PCM5102A}: BCLK, LRCK/WS, DATA, dan MCLK bila digunakan.
\end{itemize}

\subsection{Aturan packing data}
\begin{itemize}
\item \texttt{DATA\_LEFT}: \texttt{[SAMPLE\_WIDTH-1:0]} berisi sampel kiri.
\item \texttt{DATA\_RIGHT}: \texttt{[SAMPLE\_WIDTH-1:0]} berisi sampel kanan.
\item Firmware menulis pasangan kiri/kanan; sampel diteruskan ke serializer tanpa antrean FIFO.
\item Jika \texttt{MUTE=1}, keluaran serial dipaksa nol tanpa mengubah isi buffer.
\end{itemize}

\section{Alat dan Bahan}

\subsection{Perangkat keras}

\subsubsection{Board FPGA}
\textbf{Basys3} digunakan sebagai platform implementasi utama. Board ini berbasis FPGA Xilinx Artix-7 dan menyediakan sumber clock, antarmuka USB-UART, serta header PMOD untuk koneksi ke perangkat eksternal.

\subsubsection{Modul DAC audio}
\textbf{PCM5102A Audio DAC Module} digunakan sebagai penerima data I2S. Modul ini mendukung antarmuka audio serial, berbagai \textit{sample rate}, dan keluaran analog stereo untuk verifikasi suara.

\subsubsection{Perangkat bantu}
\begin{itemize}
    \item \textbf{Oscilloscope / logic analyzer / ILA} untuk memverifikasi sinyal BCLK, WS, dan DATA.
    \item \textbf{Speaker atau headphone aktif} untuk uji keluaran audio.
    \item \textbf{Breadboard dan kabel jumper} untuk koneksi Basys3 ke PCM5102A.
\end{itemize}

\subsection{Perangkat lunak}
\begin{itemize}
    \item \textbf{AMD Vivado Design Suite} untuk RTL design, synthesis, implementation, dan integrasi SoC.
    \item \textbf{Xilinx Vitis} untuk pengembangan perangkat lunak uji dalam bahasa C.
    \item \textbf{Vivado Simulator} atau simulasi lain bila digunakan untuk verifikasi RTL.
    \item \textbf{LaTeX} untuk penulisan dokumen skripsi.
\end{itemize}

\section{Metode yang Digunakan}

\subsection{Metode perancangan hardware}

Perancangan perangkat keras dilakukan dengan tahapan berikut:
\begin{enumerate}
    \item Menentukan spesifikasi I2S TX meliputi format data, lebar sampel, mode frekuensi sampling, dan pemetaan register.
    \item Mendesain arsitektur modul yang mencakup AXI4-Lite slave, register \texttt{DATA\_LEFT}/\texttt{DATA\_RIGHT} dan \texttt{CONTROL}, pembagi clock, serta serializer I2S.
    \item Mengimplementasikan RTL dalam Verilog dengan pemisahan blok sekuensial dan kombinasional secara jelas.
    \item Mengintegrasikan peripheral ke dalam Vivado IP Integrator bersama MicroBlaze V, BRAM, clocking, dan peripheral pendukung.
\end{enumerate}

\subsection{Metode verifikasi dan simulasi}

Verifikasi dilakukan pada dua level:
\begin{enumerate}
    \item \textbf{Verifikasi fungsional RTL}, untuk memastikan pembangkitan sinyal I2S, urutan bit, dan logika \texttt{DATA}/\texttt{CONTROL} berjalan sesuai spesifikasi.
    \item \textbf{Validasi hardware}, untuk memastikan desain tetap bekerja setelah synthesis dan implementation pada FPGA nyata.
\end{enumerate}

Parameter yang diamati meliputi:
\begin{itemize}
    \item relasi timing \texttt{i2s\_bclk}, \texttt{i2s\_ws}, dan \texttt{i2s\_data};
    \item kontinuitas aliran sampel pada skenario uji;
    \item akurasi frekuensi sampling nominal rendah dan tinggi;
    \item keberhasilan keluaran audio pada DAC.
\end{itemize}

\subsection{Metode integrasi SoC}

Integrasi peripheral ke dalam SoC dilakukan melalui alur berikut:
\begin{enumerate}
    \item \textbf{IP Packaging}: mengemas modul I2S sebagai custom IP.
    \item \textbf{Block Design}: menghubungkan MicroBlaze V, AXI interconnect, BRAM, Clocking Wizard, UART, dan IP I2S.
    \item \textbf{Address Allocation}: menetapkan base address peripheral agar dapat diakses dari Vitis.
    \item \textbf{Constraint Management}: memetakan pin I2S ke konektor board sesuai file \texttt{.xdc}.
\end{enumerate}

\subsection{Metode pengembangan software}

Perangkat lunak uji dikembangkan dalam bahasa C pada Vitis dengan fungsi utama:
\begin{itemize}
    \item menginisialisasi register peripheral I2S;
    \item menulis sampel audio ke register \texttt{DATA\_LEFT} dan \texttt{DATA\_RIGHT};
    \item mengaktifkan mode operasi dan memilih frekuensi sampling;
    \item menyinkronkan waktu penulisan data ke dalam register secara langsung (\textit{programmed I/O});
    \item menghasilkan sinyal uji sederhana seperti gelombang sinus atau \textit{square wave}.
\end{itemize}

\subsection{Metode pengujian hardware}

Pengujian hardware dilakukan dengan langkah berikut:
\begin{enumerate}
    \item Memprogram FPGA dengan bitstream hasil implementasi.
    \item Menghubungkan Basys3 ke PCM5102A.
    \item Menjalankan aplikasi Vitis untuk mengirim sampel audio.
    \item Mengamati sinyal I2S menggunakan ILA atau osiloskop.
    \item Memverifikasi keluaran audio melalui speaker/headphone.
\end{enumerate}

\subsection{Pengukuran dan evaluasi}

Metode evaluasi meliputi:
\begin{itemize}
    \item \textbf{Resource utilization}: LUT, FF, dan BRAM dari laporan implementasi.
    \item \textbf{Clock accuracy}: frekuensi aktual hasil Clocking Wizard dan frekuensi terukur sinyal I2S.
    \item \textbf{Fungsionalitas audio}: keberhasilan playback dan kontinuitas sampel yang memadai pada skenario uji utama.
\end{itemize}
