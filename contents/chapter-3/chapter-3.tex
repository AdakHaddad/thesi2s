\chapter{Perancangan Sistem}

\section{Spesifikasi Sistem}

Sistem yang dirancang pada penelitian ini adalah peripheral I2S TX berbasis AXI4-Lite yang diimplementasikan pada FPGA Basys3 (Artix-7 XC7A35T). Spesifikasi lengkap sistem adalah sebagai berikut:

\begin{itemize}
    \item \textbf{Protokol audio}: I2S (format Philips standar), mode transmitter (TX) saja.
    \item \textbf{Format data}: Stereo 24-bit per kanal dalam slot 32-bit (1-bit delay, MSB first, padding nol untuk bit 24--31).
    \item \textbf{Mode sampling rate}:
    \begin{itemize}
        \item Mode 0 ($\approx$48 kHz): BCLK = \texttt{audio\_clk} / 8, $F_s$ = BCLK / 64
        \item Mode 1 ($\approx$96 kHz): BCLK = \texttt{audio\_clk} / 4, $F_s$ = BCLK / 64
    \end{itemize}
    \item \textbf{Clock audio}: \texttt{audio\_clk} = 24,56897 MHz (dari MMCM Clocking Wizard, input 100 MHz).
    \item \textbf{Antarmuka kontrol}: AXI4-Lite slave, lebar data 32-bit, lebar alamat 32-bit.
    \item \textbf{Buffer}: FIFO TX internal (data stereo frame).
    \item \textbf{Interrupt}: IRQ watermark dan underrun.
    \item \textbf{Target perangkat keras}: FPGA Basys3, DAC PCM5102A melalui header Pmod.
    \item \textbf{Perangkat lunak}: Vitis \texttt{main.cpp}, \textit{bare-metal}.
\end{itemize}

\section{Perancangan Mekanisme IRQ}

Peripheral membangkitkan satu sinyal IRQ yang disambungkan ke kontroler interrupt prosesor. Kondisi pemicu IRQ:
\begin{enumerate}
    \item \textbf{IRQ Watermark}: Level FIFO turun di bawah nilai WATERMARK register, dan bit \texttt{IRQ\_EN[0]} aktif. Kondisi ini memberi sinyal kepada CPU untuk segera mengisi ulang FIFO.
    \item \textbf{IRQ Underrun}: Terjadi \textit{underrun} (FIFO kosong saat frame dibutuhkan) dan bit \texttt{IRQ\_EN[1]} aktif. Digunakan untuk mendeteksi masalah kecukupan bandwidth perangkat lunak.
\end{enumerate}

Flag IRQ bersifat W1C (\textit{Write-1-to-Clear}): perangkat lunak menghapus flag dengan menulis 1 ke bit yang bersangkutan pada register IRQ\_STATUS.

\section{Integrasi dengan Perangkat Lunak (Vitis \texttt{main.cpp})}

Alur inisialisasi dan pemutaran audio dari sisi perangkat lunak (Vitis \textit{bare-metal}):

\begin{enumerate}
    \item Inisialisasi peripheral: tulis ke CONTROL untuk reset FIFO (\texttt{FIFO\_RST=1}) dan pilih mode (\texttt{FS\_MODE}).
    \item Atur watermark: tulis nilai watermark ke register WATERMARK.
    \item Aktifkan IRQ: tulis ke IRQ\_EN.
    \item \textit{Prefill} FIFO: isi FIFO dengan beberapa frame audio sebelum mengaktifkan playback.
    \item Aktifkan playback: set \texttt{ENABLE=1} pada CONTROL.
    \item ISR (\textit{Interrupt Service Routine}): saat IRQ watermark dipicu, ISR mengisi ulang FIFO dengan frame audio berikutnya dari buffer perangkat lunak, lalu menghapus flag IRQ.
\end{enumerate}

Prosedur penggantian mode sampling rate yang aman:
\begin{enumerate}
    \item Set \texttt{ENABLE=0} (hentikan playback).
    \item Set \texttt{FIFO\_RST=1} (kosongkan FIFO).
    \item Ubah \texttt{FS\_MODE}.
    \item \textit{Prefill} FIFO kembali.
    \item Set \texttt{ENABLE=1}.
\end{enumerate}
