\section{Metode yang Digunakan}

\subsection{Metode perancangan hardware}
RTL dirancang secara modular menggunakan Verilog HDL yang memisahkan bagian datapath dan control path \citep{palnitkar_verilog_2003}. Desain difokuskan pada serializer I2S standar Philips, register data, shadow register untuk perlindungan data, serta modul clock desimal NCO agar rancangan bersifat \textit{vendor-agnostic} dan mendukung \textit{sample rate} arbitrer.

\subsection{Metode verifikasi dan simulasi}
Verifikasi dilakukan lewat simulasi testbench di Vivado Simulator untuk memastikan fungsionalitas sinyal I2S. Selanjutnya dilakukan validasi fisik dengan mengamati timing clock dan bit data serial pada osiloskop serta mendengarkan hasil reproduksi suara dari DAC.

\subsection{Metode integrasi SoC dan pengembangan software}
IP kustom dikemas menggunakan Vivado IP Packager lalu dimasukkan ke dalam Block Design untuk dihubungkan dengan MicroBlaze V melalui AXI Interconnect \citep{xilinx_axi_ref_2017}. Firmware bare-metal dikembangkan di Vitis untuk mengontrol peripheral, menginisialisasi parameter register, melakukan generator DDS software, dan mengelola sinkronisasi pengiriman data lewat status polling.

\section{Perancangan Block Design Sistem}

Perancangan Block Design dilakukan untuk menyatukan custom IP I2S ke dalam sistem SoC berbasis prosesor MicroBlaze V agar dapat diakses oleh program perangkat lunak.

\subsection{Integrasi SoC MicroBlaze V}
Vivado IP Integrator digunakan untuk menghubungkan komponen SoC: prosesor soft-core MicroBlaze V \citep{xilinx_microblaze_ref_2021}, memori lokal BRAM, Processor System Reset, AXI SmartConnect, AXI UARTLite untuk komunikasi terminal, dan custom IP I2S. Alamat dasar (Base Address) peripheral ditentukan secara otomatis oleh Vivado dan dipetakan ke dalam header file \path{xparameters.h} di Vitis.

\subsection{Perancangan Clocking Audio Berbasis NCO}
Clock audio diturunkan secara sinkron dari clock sistem AXI 100 MHz menggunakan teknik \textit{Numerically Controlled Oscillator} (NCO) desimal modulo 100.000.000 (penjelasan matematis rinci dibahas pada Bab~II). Sinyal detak yang dihasilkan meliputi \textbf{BCLK} ($64 \times F_s$) untuk serializer, serta \textbf{MCLK} ($256 \times F_s$) sebagai referensi clock DAC eksternal.

\subsection{Perancangan Register AXI4-Lite}
Akses register IP I2S oleh MicroBlaze V dilakukan melalui alamat memory-mapped (MMIO) berbasis antarmuka bus AXI4-Lite. Register internal IP terdiri dari \path{DATA_LEFT} (0x00), \path{DATA_RIGHT} (0x04), \path{CONTROL} (0x08), dan \path{STATUS} (0x0C). Peta register dan penjelasan fungsi bitfield sepenuhnya identik dengan detail perancangan yang disajikan pada Tabel~\ref{tab:ch4-regmap} dan Tabel~\ref{tab:ch4-control} di Bab~IV (halaman~\pageref{tab:ch4-regmap}).

Untuk sinkronisasi pengiriman data, firmware melakukan polling terhadap bit status \path{STATUS[0]} yang berubah logika setiap kali perangkat keras memindahkan sampel dari register AXI ke shadow register internal di awal frame I2S.

\subsection{Signal Generator dan Constraint FPGA}
Signal generator digital dibangun pada sisi software untuk menghasilkan sampel sinus atau kotak yang kemudian disalurkan ke register data peripheral. Pin eksternal I2S dipetakan ke port PMOD Basys3 menggunakan file constraint \path{.xdc} dengan standar level tegangan LVCMOS33. Setelah Block Design divalidasi, dilakukan sintesis, implementasi, generate bitstream, dan ekspor hardware platform (\path{.xsa}) ke Vitis.
