\chapter{Perancangan Sistem}

\section{Spesifikasi Sistem}

Sistem yang dirancang adalah peripheral I\textsuperscript{2}S TX berbasis AXI4-Lite dengan spesifikasi sebagai berikut:

\begin{itemize}
    \item \textbf{Platform FPGA}: Basys3 (Artix-7 XC7A35T), dikembangkan dengan Xilinx Vivado.
    \item \textbf{Mode operasi}: TX-only, format Philips I\textsuperscript{2}S, stereo 2 kanal.
    \item \textbf{Kedalaman data}: 24-bit per kanal, dikemas dalam slot 32-bit.
    \item \textbf{Mode laju sampling}:
    \begin{itemize}
        \item Mode~0 (\texttt{FS\_MODE=0}): $\approx$47,99~kHz (BCLK $\div 8$)
        \item Mode~1 (\texttt{FS\_MODE=1}): $\approx$95,97~kHz (BCLK $\div 4$)
    \end{itemize}
    \item \textbf{Clock audio}: \texttt{audio\_clk} = 24,56897~MHz (dari MMCM \textit{Clocking Wizard}).
    \item \textbf{Antarmuka kontrol}: AXI4-Lite, lebar data 32-bit, lebar alamat 8-bit.
    \item \textbf{FIFO TX}: Kedalaman 256 frame stereo (512 sampel), dengan mekanisme \textit{push}/\textit{pop} sinkron.
    \item \textbf{Interrupt}: IRQ berbasis \textit{watermark} level FIFO, mekanisme W1C (\textit{Write-1-to-Clear}) pada register \texttt{IRQ\_STATUS}.
    \item \textbf{DAC target}: PCM5102A sebagai I\textsuperscript{2}S slave.
    \item \textbf{Perangkat lunak}: Program \texttt{main.cpp} pada Xilinx Vitis, mode \textit{bare-metal}.
\end{itemize}

\section{Perancangan Clock Divider dan Mode Laju Sampling}

BCLK dibangkitkan dari \texttt{audio\_clk} menggunakan pembagi (\textit{divider}) sinkron. Nilai pembagi dikontrol oleh bit \texttt{FS\_MODE} pada register CONTROL:

\begin{itemize}
    \item \texttt{FS\_MODE=0}: Pembagi BCLK = 8 ($F_{BCLK} \approx 3{,}071\,\text{MHz}$, $F_s \approx 47{,}99\,\text{kHz}$)
    \item \texttt{FS\_MODE=1}: Pembagi BCLK = 4 ($F_{BCLK} \approx 6{,}142\,\text{MHz}$, $F_s \approx 95{,}97\,\text{kHz}$)
\end{itemize}

WS dibangkitkan dari \textit{counter} bit. Setiap 32 BCLK, WS berubah: 32 BCLK untuk kanal kiri (WS=0) dan 32 BCLK untuk kanal kanan (WS=1), total 64 BCLK per frame stereo.

\section{Perancangan FIFO TX}

FIFO TX adalah antrian siklik (\textit{circular buffer}) dengan kedalaman 256 frame stereo (setiap frame berisi satu pasang sampel kiri-kanan 24-bit). Desain menggunakan implementasi sinkron berbasis BRAM pada FPGA.

\subsection{Mekanisme Push dan Pop}

\begin{itemize}
    \item \textbf{Push}: Terjadi saat perangkat lunak menulis ke register TX\_RIGHT (setelah TX\_LEFT telah disimpan sementara). Satu frame stereo (TX\_LEFT + TX\_RIGHT) dimasukkan ke FIFO.
    \item \textbf{Pop}: Terjadi pada batas frame I\textsuperscript{2}S (saat \textit{bit counter} = 0, awal frame baru). Satu frame diambil dari FIFO dan dikirim ke serializer.
    \item \textbf{Underrun}: Jika FIFO kosong saat pop diperlukan, output serializer disetel ke nol, flag UNDERRUN diset, dan (jika IRQ\_EN diaktifkan) sinyal IRQ ditegaskan.
\end{itemize}

\section{Perancangan Interrupt (IRQ Watermark dan Underrun)}

Sinyal IRQ ditegaskan (\textit{asserted}) ketika kondisi berikut terpenuhi:

\begin{enumerate}
    \item \textbf{Watermark IRQ}: \texttt{(FIFO\_LEVEL $\le$ WATERMARK) \&\& IRQ\_EN[0]}
    \item \textbf{Underrun IRQ}: \texttt{UNDERRUN\_FLAG \&\& IRQ\_EN[1]}
\end{enumerate}

Register \texttt{IRQ\_STATUS} menyimpan status IRQ yang aktif. Perangkat lunak membersihkan IRQ dengan menulis 1 ke bit yang sesuai (\textit{Write-1-to-Clear}). Rekomendasi nilai watermark: 64 frame, memberikan sekitar 670~\textmu{}s jendela pengisian pada Mode~1 (96~kHz) sebelum terjadi underrun.

Prosedur inisialisasi dan pengoperasian peripheral oleh perangkat lunak:

\begin{enumerate}
    \item Reset FIFO: Tulis \texttt{CONTROL[FIFO\_RST]=1} (otomatis \textit{self-clearing}).
    \item Konfigurasi mode: Set \texttt{CONTROL[FS\_MODE]} ke mode yang diinginkan (harus dilakukan saat \texttt{ENABLE=0}).
    \item Set watermark: Tulis nilai ke register \texttt{WATERMARK} (mis. 64).
    \item \textit{Prefill} FIFO: Tulis sejumlah frame (mis. 200 frame) ke TX\_LEFT dan TX\_RIGHT.
    \item Aktifkan IRQ: Tulis \texttt{IRQ\_EN = 0x01} (atau \texttt{0x03} untuk IRQ underrun juga).
    \item Mulai playback: Set \texttt{CONTROL[ENABLE]=1}.
    \item Dalam ISR: Baca \texttt{IRQ\_STATUS}, isi ulang FIFO dalam batch (mis. 128 frame), bersihkan IRQ (\textit{W1C}).
\end{enumerate}

\noindent\textbf{Penting}: Perubahan \texttt{FS\_MODE} hanya aman dilakukan saat \texttt{ENABLE=0}. Untuk mengganti mode saat sedang berjalan: pause (\texttt{ENABLE=0}) $\to$ reset FIFO (\texttt{FIFO\_RST}) $\to$ ubah \texttt{FS\_MODE} $\to$ prefill ulang $\to$ play (\texttt{ENABLE=1}).
