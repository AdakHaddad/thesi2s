\hspace{1cm}This research focuses on the design of an I2S (\textit{Inter-IC Sound}) TX-only peripheral module using Verilog HDL, integrated with a MicroBlaze soft processor via an AXI4-Lite slave interface on a Basys3 Artix-7 FPGA. The designed peripheral supports 24-bit stereo audio in 32-bit slots (Philips I2S standard), two configurable sample rate modes ($\approx$48\,kHz and $\approx$96\,kHz) derived from a Vivado MMCM, a 256-frame stereo TX FIFO, a configurable watermark IRQ mechanism for efficient FIFO refill, and control/status registers accessible via AXI4-Lite read/write operations. The digital I2S output is transmitted to a PCM5102A DAC to produce stereo analog audio output.

\hspace{1cm}The research methodology includes: (1) audio clock derivation ($\approx$24.57\,MHz) using the Vivado MMCM from a 100\,MHz reference; (2) AXI4-Lite register map design with nine registers (ID, CONTROL, STATUS, WATERMARK, IRQ\_EN, IRQ\_STATUS, VOLUME, TX\_LEFT, TX\_RIGHT); (3) synchronous register-array-based TX FIFO implementation with deterministic push/pop logic; (4) I2S serializer implementation with BCLK and WS generators producing Philips-compliant I2S frames; (5) development of a \texttt{main.cpp} demo application on Xilinx Vitis (bare-metal MicroBlaze) that plays a 1\,kHz sine wave with ISR-based FIFO refill; and (6) verification through Vivado RTL simulation and hardware-in-the-loop testing using ILA and the PCM5102A DAC on the Basys3 board.

\hspace{1cm}Results demonstrate that the designed I2S TX-only peripheral operates correctly in both sample rate modes, with actual frequency error below 0.03\% of the target. The TX FIFO and watermark IRQ mechanism successfully decouple the CPU time domain from the audio domain, enabling glitch-free audio playback without underruns. ILA verification confirms that the BCLK, WS, and DATA signal format complies with the Philips I2S standard using 32-bit slots with 24-bit data. The PCM5102A analog output produces a clean 1\,kHz sine wave verified by oscilloscope measurement.
