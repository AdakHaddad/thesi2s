\chapter{Kesimpulan dan Saran}

\section{Kesimpulan}

Penelitian ini merancang dan mengimplementasikan \textbf{IP core transmitter I2S} dengan antarmuka \textbf{AXI4-Lite} pada SoC \textbf{MicroBlaze V}, board \textbf{Basys3}, dan keluaran menuju \textbf{PCM5102A}. Berdasarkan hasil pada Bab IV, dapat disimpulkan:

\begin{enumerate}
	\item IP berhasil diintegrasikan sebagai peripheral AXI4-Lite dengan peta register aktif \texttt{DATA\_LEFT (0x00)}, \texttt{DATA\_RIGHT (0x04)}, dan \texttt{CONTROL (0x08)}. Register \texttt{0x0C} disediakan sebagai cadangan untuk pengembangan lanjutan.
	\item Aplikasi Vitis bare-metal dapat mengendalikan IP melalui \textit{memory-mapped I/O} (\texttt{Xil\_Out32}/\texttt{Xil\_In32}) untuk inisialisasi kontrol, pengisian sampel kiri-kanan, dan verifikasi \textit{readback} register.
	\item Pengujian gelombang dan audio menunjukkan sinyal \texttt{BCLK}, \texttt{WS}, dan \texttt{DATA} mengikuti mekanisme \textbf{Philips I2S} (1-bit delay, 64 BCLK/frame stereo) dan mampu menghasilkan keluaran audio pada perangkat keras target.
\end{enumerate}

\section{Saran}

\begin{enumerate}
	\item Menambahkan \textbf{buffer antrian internal} untuk data sampel dan mekanisme notifikasi/interrupt agar pengiriman sampel lebih efisien dan mengurangi risiko \textit{underrun} pada beban CPU tinggi.
	\item Menyusun pemetaan register versi lanjut (misalnya status buffer dan flag error) agar transisi ke driver tingkat sistem operasi lebih terstruktur.
	\item Mengembangkan dukungan \textbf{DMA} untuk streaming audio kontinu berdurasi panjang.
	\item Menambahkan mode \textbf{RX} atau ekstensi \textbf{TDM/multi-channel} untuk kebutuhan akuisisi dan pemrosesan audio yang lebih kompleks.
	\item Melakukan karakterisasi lanjutan terhadap akurasi clock audio pada berbagai konfigurasi MMCM/Clocking Wizard.
\end{enumerate}

\noindent\textit{Penulis disarankan menyesuaikan poin kesimpulan dengan data terukur final dan arahan pembimbing.}
