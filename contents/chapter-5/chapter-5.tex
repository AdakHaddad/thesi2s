\section{Kesimpulan}

% [TODO: tulis 4–5 butir kesimpulan berdasarkan hasil di Bab IV]

Penelitian ini telah berhasil merancang, mengimplementasikan, dan
memverifikasi peripheral I2S transmitter berbasis AXI4-Lite pada FPGA
Basys3 yang terhubung ke DAC PCM5102A. Kesimpulan penelitian ini adalah:

\begin{enumerate}
    \item Modul I2S transmitter berhasil dirancang dalam Verilog dengan
          format \textbf{24-bit data / 32-bit slot per channel, stereo},
          mendukung dua mode \emph{sampling rate}: Mode~0 ($\approx$47,99 kHz)
          dan Mode~1 ($\approx$95,97 kHz), yang keduanya diturunkan dari clock
          audio 24,568\,97 MHz hasil konfigurasi MMCM pada Basys3.
          % [TODO: tambahkan nilai deviasi Fs jika ada]

    \item Integrasi modul sebagai slave \textbf{AXI4-Lite} pada SoC
          Basys3 di Vivado IP Integrator berhasil dilakukan dengan register
          map enam register 32-bit; program demo \texttt{main.cpp} di Vitis
          dapat mengonfigurasi mode, mengisi TX FIFO, dan menangani
          watermark IRQ dengan benar.
          % [TODO: tambahkan nilai utilisasi resource LUT/FF/BRAM aktual]

    \item TX FIFO berkapasitas \textbf{256 frame stereo} beserta mekanisme
          \textbf{watermark IRQ} (default threshold 64 frame, dapat
          dikonfigurasi via register FIFO\_THR) bekerja sesuai rancangan:
          CPU menerima interupsi hanya ketika level FIFO turun di bawah
          ambang, sehingga overhead polling dapat dieliminasi.

    \item Verifikasi melalui \textbf{simulasi RTL} (Vivado XSIM) menunjukkan
          waveform BCLK, WS, dan SDATA yang sesuai dengan spesifikasi
          protokol I2S Philips; tidak ada anomali \emph{glitch} atau
          pelanggaran timing yang terdeteksi.
          % [TODO: ringkaskan temuan spesifik dari simulasi]

    \item Verifikasi \emph{on-chip} menggunakan \textbf{ILA} mengonfirmasi
          frekuensi BCLK dan Fs pada kedua mode sesuai nilai teoritis
          (deviasi $<$[TODO]\%), dan verifikasi \textbf{uji audio} pada
          PCM5102A membuktikan keluaran sinyal audio yang bersih pada
          kedua mode \emph{sampling rate}.
          % [TODO: isi nilai deviasi dan kualitas audio yang terukur]
\end{enumerate}

%   5. Portasi ke platform FPGA lain (Intel/Altera Cyclone V, Lattice ECP5)
%   6. Optimasi resource: menggunakan BRAM sebagai FIFO untuk mengurangi
%      penggunaan LUT/FF
%   7. Penambahan fitur volume control atau DSP (equalizer, mixer) di

\section{Saran}

% [TODO: tulis 4–6 butir saran pengembangan lanjutan]

Berdasarkan hasil penelitian ini, beberapa saran untuk pengembangan
lanjutan adalah:

\begin{enumerate}
    \item \textbf{Penambahan DMA Controller}: Transfer data audio saat ini
          dilakukan melalui \emph{programmed I/O} yang membebani CPU.
          Integrasi DMA (misalnya AXI DMA IP dari Xilinx) akan memungkinkan
          \emph{streaming} audio berkapasitas besar dari memori ke FIFO
          tanpa intervensi CPU, sehingga CPU dapat menjalankan tugas lain
          secara paralel.

    \item \textbf{Pengembangan Linux ALSA Driver}: Untuk memperluas
          ekosistem penggunaan, peripheral I2S TX ini dapat diporting ke
          platform berbasis Zynq-7000 atau Zynq UltraScale+ yang menjalankan
          Linux, dengan mengembangkan driver ALSA (\emph{Advanced Linux
          Sound Architecture}) sehingga dapat digunakan oleh aplikasi audio
          Linux standar.

    \item \textbf{Penambahan Mode Sampling}: Penelitian ini membatasi pada
          dua mode (~48 kHz dan ~96 kHz). Pengembangan lanjutan dapat
          menambahkan mode 44,1 kHz (standar CD) melalui konfigurasi MMCM
          tambahan atau PLL terpisah, serta mode 192 kHz untuk
          \emph{high-resolution audio}.

    \item \textbf{Penambahan Jalur RX (Full-Duplex)}: Modul saat ini hanya
          mencakup transmisi (TX). Penambahan jalur penerima I2S (RX) akan
          memungkinkan aplikasi perekaman dan pemrosesan audio secara
          \emph{full-duplex}, misalnya untuk sistem \emph{voice pickup}
          dengan mikrofon digital I2S.

    \item \textbf{Optimasi Resource dengan BRAM}: Implementasi TX FIFO saat
          ini menggunakan register berbasis LUT/FF. Penggunaan BRAM
          (\emph{Block RAM}) bawaan Artix-7 sebagai media penyimpanan FIFO
          akan mengurangi konsumsi LUT secara signifikan dan memungkinkan
          kedalaman FIFO yang lebih besar.

    \item \textbf{Pemrosesan Sinyal Digital (DSP) On-chip}: Peripheral dapat
          dikembangkan dengan menambahkan blok DSP di dalam FPGA sebelum data
          dikirim ke DAC, misalnya: kontrol volume, \emph{equalizer} parametrik,
          atau \emph{sample-rate converter} untuk mendukung berbagai Fs secara
          fleksibel.
\end{enumerate}
