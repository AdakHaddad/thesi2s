Pengamatan sinyal pada \texttt{i2s\_bclk}, \texttt{i2s\_ws}, dan \texttt{i2s\_data} dilakukan menggunakan alat ukur digital (osiloskop/logic analyzer) untuk memverifikasi timing Philips I2S~\citep{philips_i2s_1986}.

\begin{itemize}
    \item Satu frame stereo terdiri dari tepat 64 periode BCLK (32 kiri + 32 kanan).
    \item WS beralih satu BCLK \textit{sebelum} MSB data, sesuai spesifikasi Philips.
    \item Bit pertama setelah setiap peralihan WS selalu bernilai 0.
\end{itemize}

\section{Identifikasi dan Perbaikan Bug Firmware}

Selama validasi perangkat keras ditemukan beberapa bug yang bersumber dari \textbf{kode firmware} (\texttt{helloworld.c}), bukan dari RTL.

\subsection{Bug 1: Masker Merusak Bit Tanda Sampel}
\textbf{Gejala}: Amplitudo gelombang tidak seimbang, waveform terdistorsi.
\textbf{Solusi}: Menghapus penggunaan masker bit dan menyesuaikan geseran agar MSB int16 tepat di bit \texttt{[g\_width-1]}.

\subsection{Bug 2: Timing Loop Berbasis usleep()}
\textbf{Gejala}: Frekuensi audio terukur lebih rendah dari target.
\textbf{Penyebab}: Fungsi \texttt{usleep()} memiliki granularitas yang rendah dan tidak memperhitungkan waktu eksekusi instruksi di dalam loop, sehingga terjadi akumulasi keterlambatan (\textit{drift}).
\textbf{Solusi}: Mengganti \texttt{usleep()} dengan loop berbasis \textit{deadline} menggunakan CSR \texttt{mcycle} RISC-V untuk presisi tinggi.

\begin{lstlisting}[language=C, caption={Implementasi pembacaan CSR \texttt{mcycle} dan loop \textit{deadline}.}, label={lst:ch5-timing-fix}]
// Membaca counter 64-bit pada MicroBlaze V
static inline uint64_t rdcycle64(void) {
    uint32_t lo, hi, hi2;
    do {
        __asm__ volatile ("rdcycleh %0" : "=r"(hi));
        __asm__ volatile ("rdcycle  %0" : "=r"(lo));
        __asm__ volatile ("rdcycleh %0" : "=r"(hi2));
    } while (hi != hi2);
    return ((uint64_t)hi << 32) | lo;
}

// Loop utama dengan target waktu absolut
uint64_t t_deadline = rdcycle64();
while (1) {
    stream_sample();
    t_deadline += period_cycles;
    while (rdcycle64() < t_deadline) {} // Menunggu deadline
}
\end{lstlisting}

Dengan menggunakan CSR \texttt{mcycle}, sistem memperoleh resolusi waktu sebesar 10 ns pada clock 100 MHz. Penggunaan target waktu absolut (\texttt{t\_deadline}) memastikan bahwa setiap sampel dikirim pada interval yang tepat, sehingga mengeliminasi \textit{drift} dan menghasilkan frekuensi audio yang akurat.

\subsection{Penjelasan variabel dan fungsi kunci firmware}

Implementasi pada berkas \texttt{helloworld.c} menggunakan beberapa komponen perangkat lunak penting untuk menjamin fidelitas audio:
\begin{itemize}
    \item \textbf{phase}: Akumulator fase 32-bit yang menentukan posisi pada tabel sinus. Indeks tabel diambil dari bit teratas (\texttt{phase >> 27}).
    \item \textbf{step}: Nilai penambahan fase per iterasi, dihitung berdasarkan rasio frekuensi target nada terhadap laju sampling (\texttt{Fs}).
    \item \textbf{rdcycle64()}: Fungsi pembacaan CSR \texttt{mcycle} RISC-V dengan deteksi \textit{rollover} untuk mendapatkan waktu sistem 64-bit yang presisi.
    \item \textbf{stream\_sample()}: Fungsi yang melakukan \textit{bit-scaling} (penggeseran sampel 16-bit ke 24-bit) dan pengiriman data ke register hardware melalui MMIO.
    \item \textbf{apply\_control()}: Menggabungkan status perangkat lunak (\texttt{enable, mute, fs\_family}) ke dalam register kontrol hardware melalui operasi bitwise.
\end{itemize}
