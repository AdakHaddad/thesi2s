\chapter{Tinjauan Pustaka dan Dasar Teori}

\section{Tinjauan Pustaka}

Bagian ini membahas penelitian-penelitian terdahulu yang relevan dengan perancangan peripheral I2S TX-only pada FPGA dengan antarmuka AXI4-Lite, serta penggunaan DAC PCM5102A sebagai perangkat penerima audio digital.

\subsection{Penelitian tentang Implementasi I2S pada FPGA}

Zhang dan Liu (2019) mengembangkan IP core I2S transmitter dan receiver untuk aplikasi audio processing pada Xilinx Zynq-7000. Penelitian mereka mencapai latency di bawah 50 mikrodetik dengan dukungan laju sampel hingga 96\,kHz. Implementasi menggunakan dual-port BRAM sebagai \textit{audio buffer} untuk pemutaran yang \textit{smooth}. Kontribusinya adalah \textit{testbench} komprehensif dengan pengukuran kualitas audio (SNR, THD). Keterbatasannya adalah ketergantungan pada Zynq Processing System (PS) untuk operasi DMA, tidak sepenuhnya pada Programmable Logic (PL) saja.

Anderson dan Lee (2021) merancang antarmuka I2S untuk array mikrofon digital pada FPGA. Penelitian ini berfokus pada I2S receiver multi-channel dengan dukungan TDM (\textit{Time Division Multiplexing}) hingga 8 kanal. Keterbatasannya adalah hanya mendukung mode penerimaan, tidak termasuk kemampuan transmisi.

Tanaka et al.\ (2020) mengimplementasikan I2S \textit{codec} dengan konversi laju sampel (\textit{sample rate conversion}) pada Lattice FPGA menggunakan \textit{polyphase filter}. Kontribusinya adalah konversi audio berkualitas tinggi antara 44,1\,kHz dan 48\,kHz. Namun, kompleksitas operasi DSP memerlukan blok DSP yang signifikan.

\subsection{Penelitian tentang Peripheral AXI4-Lite pada Xilinx}

Sharma dan Kumar (2021) mengimplementasikan custom peripheral untuk MicroBlaze menggunakan antarmuka AXI4-Lite. Penelitian ini memberikan panduan pengembangan peripheral termasuk desain peta register, penanganan interupsi, dan pengembangan software driver. Kontribusinya adalah templat yang dapat digunakan ulang (\textit{reusable}) untuk berbagai jenis peripheral berbasis AXI4-Lite.

Rodriguez et al.\ (2022) membandingkan performa MicroBlaze klasik dan MicroBlaze V pada aplikasi sistem tertanam. Hasil penelitian menunjukkan bahwa MicroBlaze V memiliki \textit{code density} 30\% lebih baik dan performa per MHz yang sebanding.

\subsection{Penelitian tentang DAC PCM5102A}

PCM5102A dari Texas Instruments merupakan DAC stereo 32-bit yang mendukung format I2S standar dengan laju sampel antara 8\,kHz hingga 384\,kHz. Sejumlah proyek hobbyist dan penelitian akademik telah menggunakan modul PCM5102A bersama FPGA (Artyomov, 2020; Digilent Forum, 2021) untuk pemutaran audio digital. Keunggulannya termasuk kemudahan integrasi (tidak memerlukan konfigurasi I2C/SPI, semua mode dipilih melalui pin), ketersediaan modul siap pakai dengan harga terjangkau, dan kualitas audio yang memadai untuk verifikasi sinyal uji.

\subsection{Gap Penelitian dan Kontribusi}

Berdasarkan tinjauan pustaka di atas, beberapa gap yang teridentifikasi adalah:

\begin{enumerate}
\item Belum ada referensi terbuka yang mendokumentasikan secara lengkap perancangan peripheral I2S TX-only dengan antarmuka AXI4-Lite pada papan Basys3 (Artix-7), mencakup TX FIFO, IRQ \textit{watermark}, dan dua mode laju sampel.
\item Penelitian yang ada umumnya menggunakan Zynq (dengan ARM PS) atau tidak mendokumentasikan parameter konfigurasi MMCM secara detail untuk derivasi \textit{clock} audio dari osilator 100\,MHz.
\item Tidak ada panduan terstruktur untuk verifikasi peripheral I2S pada perangkat keras menggunakan ILA Vivado sekaligus keluaran audio nyata pada PCM5102A.
\end{enumerate}

Penelitian ini berkontribusi dengan:
\begin{enumerate}
\item Implementasi lengkap peripheral I2S TX-only dalam Verilog dengan antarmuka AXI4-Lite, TX FIFO 256 frame, dan IRQ \textit{watermark} pada Basys3.
\item Dokumentasi derivasi \textit{clock} audio menggunakan MMCM Vivado dari 100\,MHz ke $\approx$24,56897\,MHz beserta analisis kesalahan frekuensi aktual.
\item Aplikasi demo \texttt{main.cpp} pada Vitis yang mendemonstrasikan pemutaran sinyal uji (sinyal sinus 1\,kHz) dengan mekanisme pengisian ulang FIFO berbasis IRQ.
\item Prosedur verifikasi menggunakan ILA Vivado dan DAC PCM5102A yang terdokumentasi, termasuk panduan pengambilan \textit{screenshot} untuk kebutuhan skripsi.
\end{enumerate}

\section{Dasar Teori}

\subsection{Protokol Komunikasi I2S}

I2S (\textit{Inter-IC Sound}) adalah protokol bus serial untuk transmisi data audio digital, dikembangkan oleh Philips pada tahun 1986. Protokol ini didesain khusus untuk komunikasi antar komponen audio dalam perangkat elektronik konsumer dan profesional.

\subsubsection{Sinyal I2S}

I2S menggunakan tiga sinyal utama:
\begin{itemize}
\item \textbf{BCLK} (\textit{Bit Clock} / SCK): Sinyal klock serial untuk transmisi data. Frekuensinya adalah $F_\text{BCLK} = F_s \times N_\text{bit} \times 2$, di mana $F_s$ adalah laju sampel dan $N_\text{bit}$ adalah kedalaman bit per kanal. Untuk 24-bit dalam slot 32-bit stereo: $F_\text{BCLK} = F_s \times 64$.
\item \textbf{WS} (\textit{Word Select} / LRCLK): Menentukan kanal yang sedang dikirimkan. WS = 0 untuk kanal kiri (\textit{left}), WS = 1 untuk kanal kanan (\textit{right}). Frekuensinya sama dengan laju sampel $F_s$.
\item \textbf{DATA} (SD): Membawa data audio dalam format serial, MSB terlebih dahulu (\textit{MSB-first}), dengan representasi komplemen dua (\textit{two's complement}).
\end{itemize}

\subsubsection{Format Data I2S (Standar Philips)}

Format standar I2S Philips memiliki karakteristik utama berikut:
\begin{itemize}
\item Data dikirim MSB-\textit{first}.
\item Transisi WS terjadi \textbf{satu siklus BCLK sebelum} bit pertama data dikirimkan (\textit{one-bit delay}).
\item Slot per kanal berjumlah $N_\text{slot}$ bit; bit-bit yang tidak digunakan (setelah bit data terakhir) diisi dengan nilai 0.
\end{itemize}

Pada penelitian ini digunakan slot 32-bit per kanal dengan data 24-bit, sehingga:
\begin{itemize}
\item Bit 0 (MSB slot, tepat setelah transisi WS): \textit{padding} nol.
\item Bit 1 -- 24: data audio 24-bit, MSB ke LSB.
\item Bit 25 -- 31: \textit{padding} nol.
\end{itemize}

\subsubsection{Hubungan Frekuensi I2S}

Untuk desain ini dengan \textit{clock} audio $f_\text{audio} = 24{,}56897\,\text{MHz}$:

\begin{itemize}
\item \textbf{Mode 1 ($\approx$96\,kHz)}: BCLK = $f_\text{audio} / 4$; WS = BCLK / 64
      \[
      F_\text{BCLK} = \frac{24{,}56897}{4} \approx 6{,}142\,\text{MHz}, \quad
      F_s = \frac{6{,}142\,\text{MHz}}{64} \approx 95{,}97\,\text{kHz}
      \]
\item \textbf{Mode 0 ($\approx$48\,kHz)}: BCLK = $f_\text{audio} / 8$; WS = BCLK / 64
      \[
      F_\text{BCLK} = \frac{24{,}56897}{8} \approx 3{,}071\,\text{MHz}, \quad
      F_s = \frac{3{,}071\,\text{MHz}}{64} \approx 47{,}99\,\text{kHz}
      \]
\end{itemize}

\subsection{DAC PCM5102A}

PCM5102A dari Texas Instruments adalah konverter digital-ke-analog (\textit{DAC}) stereo 32-bit dengan karakteristik sebagai berikut:
\begin{itemize}
\item \textbf{Antarmuka}: I2S \textit{slave} (menerima BCLK, LRCLK, DIN dari master eksternal).
\item \textbf{Laju sampel yang didukung}: 8\,kHz -- 384\,kHz.
\item \textbf{Kedalaman bit}: 16, 24, atau 32 bit.
\item \textbf{Konfigurasi mode}: Tanpa bus kontrol (I2C/SPI); mode dipilih melalui pin FLT, DEMP, dan XSMT.
\item \textbf{Keluaran analog}: Stereo \textit{line-level} ($\approx$2,1\,V\textsubscript{RMS} pada beban 10\,k$\Omega$).
\item \textbf{SNR}: 112\,dB (tipikal).
\end{itemize}

\subsection{Protokol AXI4-Lite}

AXI4-Lite adalah subset dari protokol AMBA AXI4 yang disederhanakan untuk peripheral dengan kebutuhan \textit{throughput} rendah. AXI4-Lite memiliki lima kanal independen:

\begin{enumerate}
\item \textbf{Write Address Channel (AW)}: Master mengirimkan alamat dan sinyal kontrol tulis.
\item \textbf{Write Data Channel (W)}: Master mengirimkan data tulis.
\item \textbf{Write Response Channel (B)}: Slave mengirimkan respons tulis (OKAY, SLVERR).
\item \textbf{Read Address Channel (AR)}: Master mengirimkan alamat baca.
\item \textbf{Read Data Channel (R)}: Slave mengirimkan data baca dan respons.
\end{enumerate}

Setiap kanal menggunakan mekanisme \textit{handshake} VALID-READY: transfer terjadi ketika sinyal VALID dan READY keduanya tinggi pada \textit{rising edge} klock.

\subsection{FPGA Basys3 dan MMCM}

Papan Basys3 dari Digilent menggunakan FPGA Xilinx Artix-7 XC7A35T-1CPG236C dengan karakteristik utama:
\begin{itemize}
\item 33.280 sel logika, 1.800\,Kbit BRAM, 90 blok DSP48.
\item Osilator papan: 100\,MHz.
\item 5 port Pmod (4-pin dan 12-pin) untuk koneksi peripheral eksternal.
\item JTAG via USB untuk pemrograman dan ILA.
\end{itemize}

MMCM (\textit{Mixed-Mode Clock Manager}) pada Artix-7 adalah sumber daya klock khusus yang dapat menghasilkan beberapa frekuensi keluaran dari satu referensi masukan menggunakan fraksi PLL. Pada penelitian ini, Vivado Clocking Wizard dikonfigurasi untuk menghasilkan dua keluaran:
\begin{itemize}
\item \texttt{clk\_out1}: 100\,MHz (klock sistem AXI / MicroBlaze).
\item \texttt{clk\_out2}: 24,56897\,MHz (klock audio; nilai terdekat yang dapat dicapai MMCM dari target 24,576\,MHz).
\end{itemize}

\subsection{TX FIFO (\textit{First-In First-Out})}

TX FIFO pada peripheral I2S berfungsi sebagai \textit{buffer} antara domain klock CPU (AXI, 100\,MHz) dan domain klock audio (I2S, $\approx$24,57\,MHz). Dengan TX FIFO, perangkat lunak tidak perlu menulis sampel audio tepat setiap frame I2S; cukup mengisi FIFO dalam kelompok besar setiap kali level FIFO turun di bawah nilai \textit{watermark}.

\subsubsection{Parameter TX FIFO}

\begin{itemize}
\item \textbf{Lebar data}: 48-bit per entri (24-bit kanal kiri + 24-bit kanal kanan per frame stereo).
\item \textbf{Kedalaman}: 256 frame stereo.
\item \textbf{Operasi push}: Dipicu saat perangkat lunak menulis register TX\_RIGHT (setelah TX\_LEFT sudah ditulis lebih dahulu).
\item \textbf{Operasi pop}: Dipicu pada setiap batas frame I2S (awal setiap frame baru).
\item \textbf{Kondisi underrun}: Jika FIFO kosong saat pop, keluarkan nol dan set flag UNDERRUN.
\end{itemize}

\subsection{Mekanisme IRQ \textit{Watermark}}

IRQ \textit{watermark} memungkinkan CPU untuk mengisi ulang FIFO secara efisien tanpa \textit{polling}. Peripheral menghasilkan sinyal interupsi (\texttt{irq}) ketika:
\[
\text{fifo\_level} \leq \text{WATERMARK}
\]
CPU yang sedang berjalan di atas MicroBlaze mendaftarkan ISR pada AXI Interrupt Controller. Ketika IRQ \textit{watermark} terpicu, ISR mengisi FIFO hingga penuh (atau mendekati penuh), kemudian kembali ke program utama. Ini menghindari penulisan satu-per-satu sampel yang mengonsumsi seluruh bandwidth CPU.

\subsection{MicroBlaze dan Xilinx Vitis}

MicroBlaze adalah prosesor lunak (\textit{soft processor}) 32-bit dari Xilinx yang diimplementasikan pada FPGA menggunakan sumber daya logika programmable. MicroBlaze mendukung antarmuka AXI4 untuk akses memori dan AXI4-Lite untuk akses peripheral. Dalam konteks penelitian ini, MicroBlaze dikonfigurasi dengan:
\begin{itemize}
\item \textit{Clock}: 100\,MHz.
\item \textit{Local memory}: 64\,KB BRAM.
\item AXI Interrupt Controller terhubung ke output \texttt{irq} peripheral I2S.
\end{itemize}

Xilinx Vitis adalah lingkungan pengembangan perangkat lunak (\textit{IDE}) untuk target Xilinx. Pada penelitian ini, Vitis digunakan untuk:
\begin{itemize}
\item Menulis dan mengompilasi aplikasi bare-metal \texttt{main.cpp} dengan kompiler GCC untuk MicroBlaze.
\item Men-\textit{generate} BSP (\textit{Board Support Package}) dari definisi \textit{hardware} Vivado (.xsa).
\item Men-\textit{download} program ke FPGA dan melakukan \textit{debugging} via JTAG.
\end{itemize}
